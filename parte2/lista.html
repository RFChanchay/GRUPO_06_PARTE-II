Lista de Nodos

El objeto HTML DOM NodeList

Un NodeListobjeto es una lista (colección) de nodos extraídos de un documento.

Un NodeListobjeto es casi lo mismo que un HTMLCollection.

Algunos navegadores (más antiguos) devuelven un objeto NodeList en lugar de HTMLCollection para métodos como getElementsByClassName().

Todos los navegadores devuelven un objeto NodeList para la propiedad childNodes. 

La mayoría de los navegadores devuelven un objeto NodeList para el método querySelectorAll().

El siguiente código selecciona todos los <p>nodos en un documento:

Codigo 



<!DOCTYPE html>
<html>
<body>


<h2>JavaScript HTML DOM</h2>


<p>Hola mundo!</p>


<p>Hola persona!</p>


<p id="demo"></p>


<script>
const myNodelist = document.querySelectorAll("p");


document.getElementById("demo").innerHTML = "El innerHTML del segundo parrafo es: " + myNodelist[1].innerHTML;


</script>


</body>
</html>

Resultado 

                    




JavaScript HTML DOM



Hola mundo!



Hola persona!

Longitud de la lista de nodos HTML DOM

La lengthpropiedad define el número de nodos en una lista de nodos:
Codigo 



<!DOCTYPE html>
<html>
<body>


<h2>JavaScript HTML DOM</h2>


<p>Hola mundo!</p>


<p>Hola persona!</p>


<p id="demo"></p>


<script>
const myNodelist = document.querySelectorAll("p");


document.getElementById("demo").innerHTML = "El documento contiene " + myNodelist.length + " Parrafos.";


</script>


</body>
</html>

Resultado

Hola persona!

La diferencia entre una HTMLCollection y una NodeList

Una lista de nodos y una colección de HTML son prácticamente lo mismo.

Ambos son colecciones tipo matriz (listas) de nodos (elementos) extraídos de un documento. Se puede acceder a los nodos por números de índice. El índice comienza en 0.

Ambos tienen una propiedad de longitud que devuelve el número de elementos de la lista (colección).

Una HTMLCollection es una colección de elementos de documento .

Una lista de nodos es una colección de nodos de documentos (nodos de elementos, nodos de atributos y nodos de texto).

Se puede acceder a los elementos de HTMLCollection por su nombre, identificación o número de índice.

Solo se puede acceder a los elementos de NodeList por su número de índice.

Una HTMLCollection es siempre una colección en vivo . Ejemplo: si agrega un elemento <li> a una lista en el DOM, la lista en HTMLCollection también cambiará.

Una NodeList suele ser una colección estática . Ejemplo: si agrega un elemento <li> a una lista en el DOM, la lista en NodeList no cambiará.

Los métodos getElementsByClassName()y getElementsByTagName() devuelven una HTMLCollection activa.

El querySelectorAll()método devuelve una lista de nodos estática.

La childNodespropiedad devuelve una lista de nodos activa.

¡No es una matriz!

Una lista de nodos puede parecer una matriz, pero no lo es.

Puede recorrer una NodeList y consultar sus nodos por índice.

Pero no puede usar métodos de matriz como push(), pop() o join() en una lista de nodos.
